# Пакет артефактов для Claude Code

Это набор инструкций, по которым Claude Code разработает клиентскую часть
PWA-приложения кофейни «Луна Кофе» строго по ТЗ и архитектуре из ВКР.

## Куда что класть

Распакуй содержимое в КОРЕНЬ будущего проекта:

```
проект/
├── CLAUDE.md                    # главный файл правил (Claude Code читает первым)
├── docs/
│   ├── TZ.md                    # техническое задание (источник истины)
│   └── ROADMAP.md               # план по фазам
└── .claude/
    └── rules/
        ├── fsd.md               # правила Feature Sliced Design
        ├── mocks.md             # правила работы с моками
        └── conventions.md       # стиль кода и соглашения
```

`CLAUDE.md` Claude Code подхватывает автоматически. Остальные файлы он читает
по ссылкам из `CLAUDE.md`.

## Как начать работу

1. Создай пустую папку проекта.
2. Положи туда эти файлы (структуру выше).
3. Запусти Claude Code в этой папке.
4. Дай стартовый промпт (см. ниже `START_PROMPT.md`).

## Порядок разработки

Claude Code идёт по фазам из `docs/ROADMAP.md`: каркас → shared → entities →
навигация → авторизация → заказ → реал-тайм → админка → PWA → полировка →
тесты. Не давай ему перепрыгивать фазы — это сохранит порядок и качество.

## Что важно помнить тебе

- **Бэкенда нет.** Всё на моках, переключаемых флагом `NEXT_PUBLIC_USE_MOCKS`.
  Когда напишут Node.js-бэк — поменяется только тело api-функций.
- **Ассеты из Figma** ты импортируешь сам. Claude Code оставит плейсхолдеры с
  пометкой `// TODO: заменить на ассет из Figma`.
- **Это ВКР.** Решения должны быть обоснованными. Если Claude Code предлагает
  отклониться от ТЗ — пусть сначала спросит.
