# Правила Feature Sliced Design

Обязательны к соблюдению. При любом сомнении «куда положить код» — сюда.

## Слои (сверху вниз)

```
app       — инициализация: провайдеры, корневой layout, маршрутизация
pages      — страницы-композиции (на одну страницу/маршрут). Без бизнес-логики.
widgets    — крупные самодостаточные блоки UI (навбар, очередь, конструктор)
features   — пользовательские сценарии (авторизация, добавление в корзину...)
entities   — бизнес-сущности (product, order, user)
shared     — переиспользуемый код без привязки к домену
```

> Слой `pages` физически размещён в `src/views`: каталог `src/pages` занят
> Next.js под Pages Router. Алиас `@pages/*` указывает на `src/views`.

## Правило зависимостей

Слой может импортировать ТОЛЬКО из нижележащих:

- app → всё
- pages → widgets, features, entities, shared
- widgets → features, entities, shared
- features → entities, shared
- entities → shared
- shared → ничего (только внешние пакеты)

Запрещено:

- импорт вверх (entities → features)
- импорт вбок между слайсами одного слоя (feature-A → feature-B)

Если фиче нужна другая фича — подними общую логику в entities или shared, либо
оркеструй их на уровне widget/page.

## Структура слайса

Каждый слайс (модуль внутри features/widgets/entities) организован одинаково:

```
features/auth/
├── index.ts        # публичный API — ТОЛЬКО через него импортируют снаружи
├── ui/             # компоненты отображения
├── model/          # логика: хуки, стор, типы, схемы валидации
└── api/            # обращения к серверному API (сейчас — к мокам)
```

Не все папки обязательны — создавай по необходимости. Но `index.ts` обязателен.

## Публичный API (index.ts)

Снаружи слайс импортируют ТОЛЬКО через его `index.ts`:

```ts
// ✅ правильно
import { LoginForm } from '@features/auth';

// ❌ запрещено — глубокий импорт во внутренности
import { LoginForm } from '@features/auth/ui/login-form';
```

В `index.ts` реэкспортируй только то, что нужно снаружи. Внутренние детали не
экспортируй — это инкапсуляция.

## Где что лежит — шпаргалка

| Что                                 | Куда                                 |
| ----------------------------------- | ------------------------------------ |
| Кнопка, инпут, модалка (без домена) | shared/ui                            |
| formatPrice, formatDate             | shared/lib                           |
| URL API, константы                  | shared/config                        |
| Экземпляр Axios, перехватчики       | shared/api                           |
| Сторы Zustand (глобальные)          | shared/store                         |
| Тип Product, карточка товара        | entities/product                     |
| Тип Order, статусы, карточка заказа | entities/order                       |
| Форма логина, логика авторизации    | features/auth                        |
| Добавление в корзину                | features/add-to-cart                 |
| Конструктор напитка (крупный блок)  | widgets/drink-configurator           |
| Навбар                              | widgets/navbar                       |
| Страница меню (композиция)          | app/(client)/menu/page.tsx + widgets |

## Алиасы импортов

Используй алиасы, не относительные пути:

```ts
// ✅
import { Button } from '@shared/ui';
// ❌
import { Button } from '../../../shared/ui';
```

## Контроль

Правило границ проверяется eslint-plugin-boundaries. Если линтер ругается на
нарушение слоёв — это не повод отключить правило, а сигнал, что код лежит не там.
