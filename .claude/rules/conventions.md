# Соглашения и стиль кода

## Именование

- Файлы: kebab-case — `order-card.tsx`, `use-order-socket.ts`.
- Компоненты: PascalCase — `OrderCard`, `DrinkConfigurator`.
- Хуки: camelCase с `use` — `useCartStore`, `useOrderSocket`.
- Типы/интерфейсы: PascalCase — `Order`, `Product`, `UserRole`.
- Константы: UPPER_SNAKE_CASE — `MIN_PREPARATION_MINUTES`, `USE_MOCKS`.
- Сторы: `*.store.ts` — `auth.store.ts`, `cart.store.ts`.
- Схемы Zod: `*.schema.ts` — `checkout.schema.ts`.
- API: `*.api.ts` — `order.api.ts`.

## TypeScript

- `strict: true`. Никаких `any`. Неизвестное — `unknown` + сужение.
- Типы пропсов — явные. Интерфейс `Props` локально в файле компонента.
- Возвращаемые типы api-функций и хуков — явные.
- Enum статусов — через union-тип, не `enum`: `type OrderStatus = 'pending' | ...`.

## Компоненты

- Только функциональные компоненты и хуки.
- Один файл — один компонент. Вспомогательные — рядом, если приватные.
- Логику данных/состояния выноси в `model/` слайса, UI оставляй «тонким».
- Состояние, локальное для компонента (открытость модалки) — `useState`, не в
  глобальный стор.
- Селективная подписка на Zustand: `useCartStore((s) => s.items)`, не весь стор.

## Стили (Tailwind)

- mobile-first: базовые классы — мобильные, `sm:`/`md:`/`lg:` — шире.
- Цвета — через CSS-переменные темы Shadcn (`bg-primary`, `text-muted-foreground`),
  не хардкод hex в классах.
- Переключение мобильной/десктоп навигации — через `block md:hidden` /
  `hidden md:block`, без условного рендеринга на JS.

## Формы

- React Hook Form + Zod. Схема — в `model/*.schema.ts`.
- Тип формы выводится из схемы: `type X = z.infer<typeof xSchema>`. Не дублируй.
- Ошибки валидации показывай под полем.

## Сеть

- Только через `shared/api` (экземпляр Axios). Никаких прямых `fetch`/`axios` в
  компонентах.
- Обработка состояний загрузки/ошибки — в кастомном хуке, не в JSX напрямую.
- 401 обрабатывается централизованно в перехватчике, не в каждом компоненте.

## Комментарии

- На русском, по делу. Поясняй «почему», а не «что».
- Нетривиальные архитектурные решения — короткий комментарий с обоснованием
  (это ВКР, решения должны быть защищаемыми).
- `// TODO:` — для мест, ожидающих бэк или ассет из Figma.

## Git (если используется)

- Коммиты по-русски, в повелительном наклонении: «добавить конструктор напитка».
- Один коммит — одна логическая единица.
- Не коммить закомментированный код и console.log.

## Запреты

- `any`, `@ts-ignore` без крайней необходимости (если надо — с комментарием).
- Прямой доступ к localStorage для токена.
- Хардкод строк/чисел, которым место в `shared/config`.
- Новые зависимости без согласования.
- Дублирование типов между слоями.
