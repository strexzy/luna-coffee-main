# ROADMAP — порядок разработки по фазам

Claude Code идёт по фазам последовательно. Не перепрыгивай вперёд. Внутри фазы
можно выбирать порядок задач, но не выходи за её рамки без отметки о завершении.

В конце каждой фазы: прогон `type-check` + `lint`, краткий отчёт что сделано,
что осталось. Затем переход к следующей фазе.

---

## Статус разработки

Обновляется по мере прохождения фаз — быстрый ориентир для новой сессии.

- ✅ **Фаза 0** — каркас (Next.js 16, Tailwind 4, Shadcn/UI, FSD, линтинг)
- ✅ **Фаза 1** — слой shared (config, api+перехватчики, ui, lib, store, моки)
- ✅ **Фаза 2** — entities (product, order, user) на моках
- ✅ **Фаза 3** — навигация, route groups, RoleGuard, заглушки экранов
- ✅ **Фаза 4** — авторизация (формы входа/регистрации, сессия, 401, редирект по роли)
- ⏭️ **Фаза 5** — клиентский сценарий заказа (текущая, не начата)
- ⬜ Фазы 6–10 — впереди

Все изменения закоммичены и запушены в `origin/main`.

## Фаза 0 — Каркас проекта ✅

Цель: запускаемый пустой проект с настроенным стеком и структурой FSD.

- [ ] Инициализация Next.js (App Router, TypeScript strict).
- [ ] Подключение Tailwind CSS + базовый конфиг (тема, брейкпоинты).
- [ ] Установка и инициализация Shadcn/UI.
- [ ] Настройка ESLint + Prettier + правило границ слоёв (eslint-plugin-boundaries).
- [ ] Алиасы путей в tsconfig: @app, @pages, @widgets, @features, @entities, @shared.
- [ ] Каркас директорий FSD (пустые слои с index.ts).
- [ ] npm-скрипты: dev, build, type-check, lint, format.

Готово, когда: `npm run dev` поднимает пустую главную, `type-check` и `lint` чисты.

## Фаза 1 — Слой shared и базовая инфраструктура ✅

Цель: фундамент, на котором строится всё остальное.

- [ ] `shared/config` — переменные окружения, константы (URL API, брейкпоинты).
- [ ] `shared/api` — настроенный экземпляр Axios + перехватчики (заготовка).
- [ ] `shared/ui` — обёртки над Shadcn/UI: Button, Input, Card, Modal, Toast,
      Skeleton, Spinner, EmptyState.
- [ ] `shared/lib` — утилиты (formatPrice, formatDate и т.п.).
- [ ] `shared/store` — заготовки сторов Zustand (auth, cart, order).
- [ ] Мок-инфраструктура (см. `.claude/rules/mocks.md`).

Готово, когда: есть переиспользуемые примитивы и работающий мок-слой.

## Фаза 2 — Сущности (entities) ✅

Цель: бизнес-сущности предметной области.
